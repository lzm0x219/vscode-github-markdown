# ROADMAP

## 状态

- 当前版本：`v4`
- 状态：Active
- 目标：以下路线图内容均在 `v4` 内完成

## 目标

让 VS Code 的 Markdown Preview 尽可能接近 GitHub，同时保持快速、离线优先、易于维护。

## 原则

1. GitHub 一致性优先。
2. 性能优先于功能数量。
3. 扩展性优先于定制性。
4. 默认离线优先。
5. 先稳定契约，再扩展生态。

## 当前基础

已具备：

- VS Code Markdown Preview 接入
- Web Extension 兼容运行时
- 预览刷新链路
- GitHub 主题模式与主题配置
- 通过 VS Code 命令修改主题配置
- 已支持的 GitHub 风格能力：
  - 任务列表
  - 脚注
  - Alerts
  - Emoji 短代码（`:rocket:` 等）
  - Mermaid 图表 CSS 样式

## v4 范围

`v4` 聚焦三件事：

1. GitHub 兼容性
2. 可扩展架构
3. GitHub 主题支持

## 里程碑

| 里程碑          | 状态        | 重点                             | 完成标准                                                  |
| --------------- | ----------- | -------------------------------- | --------------------------------------------------------- |
| GitHub 兼容性   | In Progress | 缩小与 GitHub 的可见渲染差距     | 常见 Markdown 文档预览接近 GitHub，且已支持能力可重复验证 |
| 可扩展架构      | Planned     | 稳定插件边界与渲染契约           | 新功能可接入现有预览链路，而不需要重构整体结构            |
| GitHub 主题支持 | Implemented | 提供完整且稳定的 GitHub 主题体验 | 主题选择、系统跟随与预览一致性稳定可用                    |

## 里程碑 1：GitHub 兼容性

重点：

- GFM 基线覆盖
- 表格
- 任务列表
- 脚注
- Alerts
- 代码块表现
- 锚点行为

计划：

- 定义 GitHub Markdown 能力矩阵
- 区分 `built-in`、`extension-owned` 和 `out-of-scope`
- 收紧与 GitHub 更接近的间距、排版、代码、表格和块级样式
- 为已支持能力补最小回归检查
- 定义可重复执行的兼容性验证流程

完成标准：

- README 类文档的观感明显更接近 GitHub
- 已支持能力有清晰、诚实的文档说明
- 已支持能力具备最小回归保护

## 里程碑 2：可扩展架构

重点：

- 稳定的 `markdown-it` 插件边界
- 稳定的渲染钩子
- 最小公开扩展契约

计划：

- 明确插件职责
- 保持插件顺序显式可见
- 记录稳定的 CSS hook 与 DOM 假设
- 定义最小可用扩展接口

完成标准：

- 新增兼容性功能可以自然接入现有链路
- 内部实现细节不会泄漏到公开契约
- 架构文档能说明后续功能应该落在哪里

## 里程碑 3：GitHub 主题支持

重点：

- 亮色主题
- 暗色主题
- 高对比度主题
- 色觉辅助主题
- 单主题与系统跟随模式
- 使用 VS Code 命令修改主题配置

计划：

- 保持 GitHub 主题列表与配置项清晰可用
- 确保单主题模式与系统模式行为一致
- 保持日间与夜间主题切换稳定
- 确保主题变化后预览刷新行为可靠
- 支持通过 VS Code 命令修改主题模式与主题选择
- 为主题相关能力补最小验证与文档说明

完成标准：

- GitHub 主题选择覆盖当前已声明的主题集合
- 主题配置与预览结果保持一致
- 系统跟随模式在日间与夜间切换时行为稳定
- 主题配置可以通过 VS Code 命令稳定修改

## 非目标

本项目不打算演变为：

- CMS
- 笔记应用
- 文档数据库
- 云平台

本项目也不计划模拟以下 GitHub 在线上下文能力：

- `@mentions`
- Issue / Pull Request 引用
- 依赖仓库上下文的自动链接
- 资源上传流程

## v4 完成标准

`v4` 可视为完成，当且仅当：

- 常见 Markdown 预览明显更接近 GitHub
- 已支持能力有验证手段，也有文档说明
- 扩展边界足够稳定，便于后续继续加功能
- GitHub 主题支持稳定、清晰且与预览结果一致
